# TCREI Prompt Frame

## Purpose
TCREI is a structured prompt architecture for building high-quality reasoning workflows. It is designed for capability mastery, deep understanding, iterative refinement, and systematic folder-based knowledge organization.

TCREI = Task + Context + Reference + Embodied Roleplay + Iterate/Evaluate

---

# Core Structure

```text
[T] TASK
Define exactly what must be produced.
- What is the objective?
- What is the scope?
- What output format is required?
- What constraints apply?

[C] CONTEXT
Provide the surrounding system conditions.
- Why does this matter?
- What environment or domain does it belong to?
- What assumptions, boundaries, variables, stakeholders, or prior knowledge exist?
- What stage of the workflow is this?

[R] REFERENCE
Supply the models, examples, frameworks, or source material to anchor the reasoning.
- Existing documents
- Frameworks
- Taxonomies
- Prior outputs
- Desired style or benchmark examples

[E] EMBODIED ROLEPLAY
Assign a role, mindset, or cognitive frame.
- What type of expert is speaking?
- What disciplinary perspective should dominate?
- What level of rigor, terminology, and abstraction is required?
- What hidden assumptions should this role challenge?

[I] ITERATE + EVALUATE
Require the output to critique itself and improve.
- What are the strengths and weaknesses?
- What is missing?
- What alternative interpretations exist?
- What should be refined in the next version?
- What metrics define a better result?
```

# Master Capability Template

```text
[TASK]
Build / analyze / explain / compare / design:
{Insert specific task}

Required deliverables:
- {Deliverable 1}
- {Deliverable 2}
- {Deliverable 3}

Output format:
- Table / framework / layered analysis / workflow / matrix / folder structure

Constraints:
- No vague language
- Distinguish facts, hypotheses, interpretations
- Use systems thinking and causal logic
- Include feedback loops, emergence, nonlinear dynamics, and constraints

[CONTEXT]
Domain:
{Insert domain}

Purpose:
{Insert why this analysis matters}

System boundaries:
- Included: {Included variables}
- Excluded: {Excluded variables}

Current stage:
{Exploration / diagnosis / synthesis / implementation / review}

Prior knowledge:
{Insert relevant history, previous models, previous outputs}

[REFERENCE]
Use the following as anchors:
- Frameworks: {SWOT, 5 Whys, FMEA, Second-Order Thinking, ESG, etc.}
- Source material: {documents, examples, reports}
- Desired benchmark: {style, depth, structure}

[ROLEPLAY]
Act as:
“A multidisciplinary systems scientist and capability architect with expertise in {fields}.”

Responsibilities:
- Translate complexity into structure
- Identify hidden assumptions
- Separate signal from noise
- Connect micro-level mechanisms to macro-level outcomes
- Explain causal chains and feedback loops

Thinking style:
- First-principles reasoning
- Systems dynamics
- Multi-scale analysis
- Comparative evaluation
- Iterative refinement

[ITERATE + EVALUATE]
After producing the first output:
1. Evaluate completeness from 0–10.
2. Identify missing variables, assumptions, or contradictions.
3. Generate a second version with improvements.
4. Compare Version 1 vs Version 2.
5. Recommend the next layer of analysis.
```

# Example TCREI Applied to “Understand Thinking”

```text
[TASK]
Build a workflow architecture for understanding thinking.

[CONTEXT]
The objective is to construct a layered model of how thinking emerges, operates, fails, and improves across human, organizational, and AI systems.

[REFERENCE]
Use:
- Cognitive science
- Systems thinking
- Mental models
- Feedback loops
- CEO Thinking Toolkit
- SWOT, FMEA, 5 Whys, Second-Order Thinking

[ROLEPLAY]
Act as a senior systems architect integrating neuroscience, psychology, AI, and organizational strategy.

[ITERATE + EVALUATE]
After the architecture is built:
- Identify blind spots
- Identify missing layers
- Evaluate whether the architecture explains both individual and collective thinking
- Revise the architecture for greater explanatory power
```

# Compressed TCREI One-Line Prompt Formula

```text
Task: {what to do}
Context: {why + where + boundaries}
Reference: {frameworks/examples}
Roleplay: {expert perspective}
Iterate/Evaluate: {critique, score, improve, compare}
```

Example:

```text
Task: Build a meta-capability workflow for organizational purpose.
Context: The workflow must connect human motivation, institutional incentives, and long-term value creation.
Reference: Use systems thinking, ESG, 5Ps, SWOT, and Second-Order Thinking.
Roleplay: Act as a cross-disciplinary strategist and systems scientist.
Iterate/Evaluate: Score the result, identify missing variables, produce a revised version, and propose the next analytical layer.
```
